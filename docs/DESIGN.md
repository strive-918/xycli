# Design: XYCLI TypeScript CLI Agent

> Generated by Planner Agent | Stage 2 of software-development workflow

## 1. Module Overview

XYCLI is implemented as a TypeScript npm monorepo with npm workspaces. The design separates terminal command handling, agent orchestration, tool execution, provider adapters, MCP integration, plugin loading, and session persistence into focused packages while keeping a single local-process runtime.

Workspace layout:

| Package | Responsibility |
|---|---|
| `packages/cli` | `xycli` executable, command routing, terminal rendering, prompts, help, and exit-code mapping. |
| `packages/core` | Agent loop, planning state, context manager, config manager, permission engine, retries, circuit breaker, and shared domain types. |
| `packages/tools` | Built-in file, terminal, search, web, git, diff, and undo tools. |
| `packages/providers` | Anthropic, OpenAI, and generic OpenAI-compatible provider adapters. |
| `packages/mcp` | MCP stdio client, server lifecycle, discovery, and MCP tool bridge. |
| `packages/plugins` | Plugin manifest validation, lifecycle, command extension, and tool registration. |
| `packages/session` | SQLite store, JSONL recovery fallback, migrations, session queries, and rollback records. |

### 4.3 Idempotent Tool Execution

Every tool call stores an idempotency key composed from `sessionId`, tool name, canonical JSON input, working directory, and permission-relevant config version. If a retry observes an existing completed call with the same key, XYCLI returns the persisted result instead of repeating the side effect. File writes additionally require `expectedSha256` or a recorded pre-image hash before mutation.

## 6. Provider Adapter Design

```ts
export interface IProvider {
  name: "anthropic" | "openai" | "generic-openai";

  temperature: number;
  maxOutputTokens: number;
  metadata: Record<string, string>;
}

export interface ProviderResponse {
  message: ProviderMessage;
  toolCalls: NormalizedToolCall[];
  usage: TokenUsage;
  finishReason: "stop" | "tool_calls" | "length" | "content_filter" | "error";
}

export type ProviderStreamEvent =
  | { type: "text_delta"; text: string }
  | { type: "tool_call_delta"; call: Partial<NormalizedToolCall> }
  | { type: "usage"; usage: TokenUsage }
  | { type: "error"; error: ProviderErrorPayload }
  | { type: "done"; response: ProviderResponse };

export interface AnthropicProvider extends IProvider {
  name: "anthropic";
}

export interface OpenAIProvider extends IProvider {
  name: "openai";
}

export interface GenericProvider extends IProvider {
  name: "generic-openai";
}
```

Provider reliability policy:

| Condition | Behavior |
|---|---|
| `429`, `500`, `502`, `503`, network reset | Retry up to 2 times with exponential backoff and jitter. |
| Two consecutive transient failures on primary | Open circuit breaker and switch to fallback provider for the session. |
| Provider returns malformed tool call | Raise `ProviderError` with exit code `4`; do not execute tools. |
| Stream interruption after tool call emitted | Persist partial stream event, mark request retryable only if no tool was executed. |

## 7. Session Schema

SQLite is the canonical schema. Timestamps are ISO-8601 UTC strings. JSON values are stored as text containing canonical JSON.

  | { kind: "allow"; reason: string }
  | { kind: "deny"; reason: string }
  | { kind: "prompt"; approval: ApprovalRequest };

Decision flow:

```
ActionRequest
  → deny rules
  → secret redaction scan
  → read-only mode check
  → allow rules
  → risk classification
  → ApprovalGate
  → allow / deny / prompt
```

## 10. Context Window Management

Context assembly is deterministic:

1. Include system instructions, user goal, active plan, and permission mode.
2. Include project and user instructions after size validation.
3. Include recent messages and the last `maxRecentToolResults` tool summaries.
4. Include active file diffs and selected file excerpts ranked by explicit mention, search hits, git status, and recency.
5. Estimate tokens before the provider request.
6. If estimated input exceeds `maxInputTokenRatio` of model context, compact older messages and tool results into a persisted summary.
7. Never inject identical file content twice in one request.

```ts
export interface ContextBundle {

  abstract exitCode: 1 | 2 | 3 | 4 | 5;
  abstract retryable: boolean;
  details: Record<string, unknown>;
}

export class UserError extends XycliError {
  code = "USER_ERROR";
  exitCode = 1 as const;
  retryable = false;
}

export class ValidationError extends XycliError {
  code = "VALIDATION_ERROR";
  exitCode = 2 as const;
  retryable = false;
}

export class PermissionError extends XycliError {
  code = "PERMISSION_DENIED";
  exitCode = 3 as const;
  retryable = false;
}

export class ProviderError extends XycliError {
  code = "PROVIDER_ERROR";
  exitCode = 4 as const;
  retryable = true;
}

export class ToolError extends XycliError {
  code = "TOOL_ERROR";
  exitCode = 5 as const;
  retryable = false;
}

export class ConfigError extends ValidationError {
  code = "CONFIG_ERROR";
}
```

Exit-code mapping:

| Exit Code | Meaning |
|---:|---|
| 0 | Success |
| 1 | General user or runtime error |
| 2 | Validation or config error |
| 3 | Permission denied |
| 4 | Provider error |
| 5 | Tool error |

Recovery policy:

| Failure | Behavior |
|---|---|
| Provider transient error | Retry, then circuit-break and fail over. |
| Tool timeout | Abort tool, persist partial output, mark call failed. |
| CLI interrupt | Stop new tool dispatch, abort active work, flush session. |
| SQLite write error | Append JSONL recovery event, surface warning, continue only for read-only commands. |
| Plugin load error | Mark plugin failed and continue startup. |
