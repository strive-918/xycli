# XYCLI 产品需求文档

> 当前技术基线：v0.2.0 使用 Rust 核心库与 Rust CLI，旧 TypeScript 实现已经退役。

## 1. 产品概述

XYCLI 是面向开发者的终端原生 AI 编程 Agent。用户通过自然语言描述目标，XYCLI 负责理解仓库、制定下一步动作、调用本地或扩展工具、验证结果并保存可审计会话。

产品原则：

- 本地优先，用户始终拥有代码和会话数据；
- Provider 可替换，不绑定单一模型厂商；
- 默认权限保守，副作用明确可审计；
- 核心接口稳定，支持后续 MCP 与插件扩展；
- 出错时给出明确状态、退出码和恢复方式。

## 2. 目标用户

- 需要在终端中完成日常编码、调试和测试的开发者；
- 需要自定义模型网关或 Provider 的团队；
- 希望审计 AI 工具调用和文件变化的用户；
- 需要通过 MCP 或插件扩展内部工具的团队。

## 3. 用户故事

1. 用户可以用一句自然语言任务让 Agent 检查和修改仓库。
2. 用户可以看到模型调用了哪些工具以及工具结果。
3. 用户可以选择 Anthropic、DeepSeek、OpenAI 或兼容网关。
4. 用户可以中断长时间请求，并保留当前会话状态。
5. 用户可以继续历史会话，而不是每次丢失上下文。
6. 用户可以在执行副作用前配置权限或进行审批。
7. 用户可以查看 Git 差异并撤销 Agent 产生的变更。
8. 用户可以通过 MCP 和插件接入团队工具。
9. 用户可以诊断配置、Provider 和环境问题。
10. 用户可以通过 Cargo 或预编译二进制在主流平台安装和运行 XYCLI。

## 4. 功能需求

### FR1：Agent 循环

- 支持观察、模型推理、工具执行和结果反馈循环；
- 支持最大轮次、中断、完成、未完成和错误终态；
- 每轮状态必须持久化。

### FR2：工具系统

- 工具通过统一接口注册；
- 同时提供模型可见 Schema 和运行时校验；
- 统一处理超时、中断、错误和审计信息。

### FR3：文件操作

- 支持读取、写入、补丁和后续删除；
- 默认限制在工作区；
- 写入记录前后哈希和差异；
- 防止并发覆盖与符号链接逃逸。

### FR4：终端命令

- 使用可执行文件与参数数组，不拼接 shell；
- 默认只允许安全白名单；
- 返回 stdout、stderr、退出码、信号和超时信息；
- 高风险命令需要显式权限或审批。

### FR5：搜索与 Web

- 支持工作区文本搜索；
- 支持受网络策略控制的网页抓取；
- 对结果大小和域名进行限制。

### FR6：多模型支持

- 支持 Anthropic、DeepSeek、OpenAI 和 OpenAI 兼容接口；
- Provider 通过统一协议接入；
- 支持重试、熔断和 fallback。

### FR7：流式输出

- 流式显示助手文本、工具状态和命令输出；
- 支持无颜色和非流式模式；
- 输出层不直接承担业务状态逻辑。

### FR8：会话管理

- 保存消息、工具调用、用量和状态；
- 支持创建、继续、列表和查看；
- 中断或崩溃后可以恢复。

### FR9：MCP 协议

- 作为 MCP 客户端连接 stdio 服务；
- MCP 工具必须经过同一权限、校验和审计流程。

### FR10：自定义工具与插件

- 插件通过清单声明入口和权限；
- 加载前校验清单；
- 插件不能绕过统一工具执行边界。

### FR11：配置系统

优先级：

```text
CLI 参数 > 环境变量 > 项目配置 > 用户配置 > 内置默认值
```

配置错误返回路径明确的校验信息。密钥只通过环境变量或安全引用提供。

### FR12：上下文窗口管理

- 估算 Token；
- 选择相关文件片段；
- 达到阈值前压缩历史；
- 保留关键计划、工具结果和用户约束。

### FR13：自定义指令

- 支持用户级和项目级指令；
- 明确加载顺序和冲突规则；
- 不允许项目指令提升运行权限。

### FR14：权限与安全

- 提供 `read-only`、`auto-safe`、`full-access`；
- 文件、命令、网络、MCP 和插件都进入统一策略；
- 敏感内容在日志、会话和遥测前脱敏。

### FR15：撤销与回滚

- 记录 Agent 造成的文件变更；
- 通过哈希确认文件未被用户再次修改；
- 哈希不一致时拒绝覆盖并提示冲突。

### FR16：Git 集成

- 提供专用 status 和 diff 工具；
- 区分工作区、暂存区和会话范围；
- 不自动提交或推送，除非用户明确要求。

### FR17：遥测与诊断

- 遥测默认关闭；
- 只发送经过清洗的错误元数据；
- 提供本地 `doctor` 命令检查环境和配置。

### FR18：CLI 打包与命令

- 提供可全局调用的 `xycli` 原生二进制；
- 支持 run、resume、session、config、doctor、diff、undo 和 plugin 等目标命令；
- 支持 Cargo 源码安装和主流平台预编译归档；
- 发布产物提供 SHA-256 校验和与来源证明。

## 5. 非功能需求

### 安全

- 默认配置不得执行任意 shell；
- 工作区越界必须默认拒绝；
- Provider 密钥不得写入项目配置或普通日志。

### 可靠性

- 会话写入不能产生半截 JSON；
- 中断后两秒内尽力持久化状态；
- 失败必须返回非零退出码。

### 性能

- 缺少 Key 或参数非法时应在一秒内失败；
- 工具输出和文件读取必须有大小上限；
- 长任务不应无限增长内存或上下文。

### 可维护性

- Rust 格式检查和 Clippy 以警告即错误运行；
- Provider、Tool、SessionStore 使用接口隔离；
- 安全边界必须有自动化测试。

## 6. 当前范围

M1 已覆盖 FR1–FR4、FR6 的部分能力、FR8 的 JSON 基础能力、FR14 的最小权限边界和 FR18 的 Cargo 本地安装基础能力。

其余需求由 M2–M10 分阶段实现。目标文档不能被当作当前完成证明。

## 7. 暂不包含

- 云端 XYCLI 账号系统；
- 多人实时协作；
- 托管式代码执行环境；
- 自动绕过操作系统权限；
- 未经用户授权的 Git 提交、推送或发布。

## 8. 术语

| 术语 | 含义 |
| --- | --- |
| Agent | 驱动模型与工具循环的本地执行器 |
| Provider | 模型厂商或兼容 API 适配器 |
| Tool | 受 Schema、权限和审计约束的能力 |
| Session | 一次可持续或可恢复的任务上下文 |
| MCP | 模型上下文协议 |
| REPL | 终端中的持续交互循环 |
